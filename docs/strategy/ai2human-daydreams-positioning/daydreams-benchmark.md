# Daydreams Benchmark

Daydreams presents itself as an agentic commerce infrastructure stack rather than a single application.

## What They Do Well

- Start from a large thesis: agents become economic actors.
- Turn the thesis into infrastructure nouns: markets, routing, settlement, discovery, identity.
- Give developers a clear entry point through docs, SDKs, examples, and API surfaces.
- Keep product surfaces connected: Lucid Agents, Taskmarket, Dreams Router, xGate.
- Use protocol language such as x402, A2A, ERC-8004, agent cards, skills, entrypoints, and routing.

## How ai2human Should Differ

Daydreams helps agents buy digital work, APIs, compute, and other agent services.

ai2human helps agents buy trusted human execution when work requires identity, judgment, proof, or real-world action.

```text
Daydreams: agent commerce infrastructure
ai2human: human execution, proof, and settlement infrastructure
```

## Useful Borrowing

- Write like infrastructure, not like a task board.
- Give every module a name and responsibility.
- Show live system signals instead of only abstract claims.
- Make the developer path visible: docs, task intake API, proof schema, agent skill.
- Make the trust path visible: operator identity, proof bundle, verification gate, settlement unlock.
