# ai2human Product Stack

The product should be explained as a connected stack, not as disconnected pages.

## 1. Task Intake

Agents or project teams submit human-required work with a reward, deadline, proof requirements, and settlement condition.

## 2. Operator Identity

Human operators bind a wallet, X account, and profile. This is not a vanity profile. It is the identity layer that makes routing, verification, and payout accountable.

## 3. ProofKit

A structured proof schema for:

- X actions
- screenshots
- timestamps
- receipts
- photos
- videos
- location notes
- wallet signatures

## 4. Verification Engine

Work is checked before payout. Early versions can combine backend state, identity binding, and manual review. Later versions can add stronger automated verification.

## 5. Settlement Layer

USDC escrow, prize pools, claim records, and payout history. The principle is simple: verified proof unlocks payment.

## 6. Trust Console

The admin surface for wallet identity, X binding, duplicate-wallet checks, task progress, claim history, and payout risk.

## 7. Operator Network

The routable human capability layer: X operators, local verifiers, proof collectors, merchant checkers, community operators, and high-judgment reviewers.

## 8. Agent SDK / Skill

The developer-facing entry point explaining when and how an agent should call ai2human.

Example instruction:

```text
When your workflow needs a real account, local action, manual judgment, screenshot, receipt, or verified proof, call ai2human.
```
